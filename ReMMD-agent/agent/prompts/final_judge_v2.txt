LEVEL-1 VERDICT (single label, choose EXACTLY ONE of {level1_choices})
================================================================================
{level1_doc}

LEVEL-1 OPERATIONAL DEFINITIONS (memorise — this is the ground truth):

  True
    The post's core claim AND ALL of its supporting details are accurate.
    Zero distortions of any kind. The audience would form an entirely correct understanding.
    ⇒ Empty `level2_taxonomy` is REQUIRED.

  Mostly True
    The core claim is true and the post's main conclusion is correct.
    Exactly ONE OR TWO LOCAL non-critical issues (a minor wording mismatch, a small over-statement, a small detail off by a unit).
    The audience still forms the CORRECT overall understanding even if a small fact is off.
    ⇒ Typically 1, at most 2, MILD L2 labels (T2, T3, V2 with only a minor overlay/legend, or C2 minor frame drift).
    ⇒ NEVER use Mostly True when T1 or V1 fires (fabrication of facts or visuals immediately disqualifies).
    ⇒ NEVER use Mostly True when 3 or more L2 labels fire.

  Mixture
    BOTH a substantial true core AND a substantial misleading core co-exist; neither dominates.
    Canonical signature: post text reports a real event but the IMAGE is from a different real event ("image-from-different-event" → T3 + V2 + C2 / C1).
    Removing the true part leaves a still-misleading post; removing the false part still leaves a meaningful true core.
    ⇒ Typically 2–4 L2 labels, often the T3 + V2 + C2 cluster.

  Mostly False
    The post's main conclusion is wrong — the audience would form the WRONG overall understanding.
    Real elements exist but they are SUBORDINATE; they don't rescue the core claim.
    ⇒ Typically triggered by T1 (fabrication of facts) alone, or V1 (fabrication of visuals), or T2 + C1 jointly invalidating the core.
    ⇒ Usually 2–4 L2 labels including ONE of {T1, V1} or the joint {T2, C1}.

  False
    The core claim is fabricated / unfounded / directly contradicted; the post has NO real basis for its central message.
    ⇒ Usually triggered by T1 AND V1 together (both facts and visuals fabricated), OR T1 with 3+ companions, OR V1 with 3+ companions.
    ⇒ Typically 4–6 L2 labels, including BOTH a fabrication label (T1 or V1) and multiple cross-modal / editing labels.

================================================================================
LEVEL-2 DISTORTION TAXONOMY (multi-label; emit ZERO OR MORE of {level2_choices})
================================================================================
{level2_doc}

LEVEL-2 DECISION GUIDE — decide each label INDEPENDENTLY (yes / no):

  T1 Fabrication
    YES when the text contains a SPECIFIC invented fact (number, dollar amount, percentage, quote, person name, organisation, contract, decision, sub-event, attribution) that the evidence does NOT corroborate ANYWHERE.
    NO when the post is reporting real events even if it has adjectives or general descriptions that the evidence supports.

  T2 Distortion
    YES when text has a REAL basis but materially overstates / dramatises / understates / over-generalises / mis-attributes / shifts the conclusion-direction.
    Signal phrases that often hint at T2: "decisive", "crisis", "panic", "surge", "official confirmation", "现场大屏", "立刻", strongly-conclusive adjectives where evidence is more measured.
    NO when the post phrasing matches the evidence's certainty and magnitude.

  T3 Misleading Context
    YES when the text places REAL content into a WRONG temporal / spatial / event-frame / source / attribution context. Includes IMAGE-FROM-DIFFERENT-EVENT (image is real but mis-framed).
    NO when minor (1-3 day) date discrepancies or natural local-event reporting.

  V1 Synthetic Visual Content
    YES when the image is wholly / partially AI-generated, CGI, illustrative-but-presented-as-real, heavily-stylised infographic / 战报海报 / official poster / marketing rendering, suspicious uncanny features (broken hands, plastic skin, AI watermark, gibberish text).
    NO for plain candid news photos OR plain data charts from official sources (BEA, Eurostat).

  V2 Visual Editing
    YES when the image starts from a real photo but visible content has been edited (overlay banner, red circle / arrow / oval, added or removed objects, retouching, before-after composite). Also fires for IMAGE-FROM-DIFFERENT-EVENT.
    NO for small corner watermarks (AP/Reuters/Getty/新华社/央视/CCTV/BBC/DW) alone, standard bottom-third broadcast captions alone, plain charts without highlights on the data, OR for in-scene physical elements (sponsorship banners physically behind the speakers, placards held by participants).
    KEY TEST: "would this element still be there if I took the same picture again?" YES → no V2 from that alone. NO → V2 YES.

  C1 Semantic Inconsistency
    YES when image and text disagree on the SEMANTIC core (different person, different entity, different event-content, different attribute).
    NO when image and text agree on what is happening even if other details drift.

  C2 Contextual Inconsistency
    YES when image and text disagree on TIME / PLACE / EVENT-FRAME / SOURCE-CONTEXT (snow when post claims summer, foreign signage for a domestic-event claim, news-outlet logos / military patches inconsistent with the post's claimed source).
    NO when image's time/place/source is consistent with the post text.

  C3 Pragmatic Inconsistency
    YES when image and text take DIFFERENT stances / sentiments / conclusions about the same event (text says "celebration", image shows mourning; text says "decisive victory", image shows somber scene).
    NO when both image and text adopt the same tone/stance.

LEVEL-2 HARD RULES (never violate):
  • Posts WITHOUT images CANNOT have V1/V2/C1/C2/C3. ONLY T1/T2/T3 are eligible.
  • Be SPECIFIC: for each YES you must be able to point to a concrete cue (an evidence_id, a sentence, or "image_k").
  • DO NOT flag T1 when a sub-claim is merely partially supported — that is T2 territory.

DATASET CALIBRATION (READ CAREFULLY — IT MATTERS)
================================================================================
ReMMDBench has 5 verdicts each contributing 20% of posts:
  True 20%, Mostly True 20%, Mixture 20%, Mostly False 20%, False 20%.

L2 base rates over 500 samples:
  V2 54%, T2 44%, C1 42%, C2 42%, T3 33%, V1 29%, T1 20%, C3 13%.

CRITICAL ANTI-BIAS RULES (do NOT default to the middle):

  • DO NOT default-predict "Mostly True". An over-predicted "Mostly True" is the #1 failure mode on this benchmark. A correct prediction breakdown over a full batch should have ROUGHLY 1 in 5 predictions at each verdict — NOT 4 in 10 at Mostly True.
  • NEVER predict "Mostly True" when ANY of these is true:
      – T1 Fabrication is in your L2 → predict at least "Mostly False".
      – V1 Synthetic Visual Content is in your L2 → predict at least "Mostly False".
      – Three or more L2 labels fired → predict at least "Mixture" (more likely Mostly False).
      – Subclaim findings show ≥ 2 CONTRADICTED → predict at least "Mixture".
  • NEVER avoid "False". "False" is the correct verdict ~20% of the time. If T1 AND V1 BOTH appear in L2 plus ≥ 2 cross-modal companions, predict False without hesitation. Do NOT downgrade to "Mostly False" just because you feel uncertain.
  • NEVER predict "True" when ANY L2 label fires. "True" requires an empty `level2_taxonomy`.
  • NEVER predict "Mixture" when the false part dominates and no substantial real core remains. Drop to Mostly False / False.
  • DO NOT inflate L2 when the post is genuinely clean. About 20% of posts have ZERO L2 labels (True). When the analyser priors are NOISY (e.g. an "official poster" with no real content errors), be willing to KEEP L2 empty AND predict True.

POSITIVE PREDICTION TARGETS (use these as a sanity check on your decisions):
  - You SHOULD predict "True" for roughly 1 in 5 posts (those with empty L2 AND all findings SUPPORTED).
  - You SHOULD predict "False" for roughly 1 in 5 posts (those with T1+V1+companions OR T1/V1+3+ severe companions).
  - You SHOULD predict "Mixture" for roughly 1 in 5 posts (canonical T3+V2+C2 image-from-different-event pattern).
  - You SHOULD predict "Mostly True" for roughly 1 in 5 posts ONLY (1–2 mild L2 labels, no T1, no V1).
  - You SHOULD predict "Mostly False" for roughly 1 in 5 posts (the dominant pattern below the "False" severity threshold).

"WHEN IN DOUBT" TIE-BREAKERS — these resolve common borderline confusions:

  Borderline True vs. Mostly True:
    Default → Mostly True if you flagged ANY L2 label, even a mild T2.
    Only stay at True when you have ZERO L2 labels AND every sub-claim is SUPPORTED AND the image obviously matches the topic.

  Borderline Mostly True vs. Mixture:
    Default → Mixture when (a) 2+ L2 labels fired, OR (b) any subclaim is CONTRADICTED.
    Mostly True is reserved for "only mild local issues that don't shift conclusion".

  Borderline Mixture vs. Mostly False:
    Default → Mostly False when ANY of: T1 in L2, V1 in L2, 4+ L2 labels, ≥2 CONTRADICTED findings.
    Mixture is for posts where removing the false part still leaves a meaningful true core.

  Borderline Mostly False vs. False:
    Default → False when T1 AND V1 BOTH fired AND 2+ companion labels (4+ L2 total).
    Mostly False is for "the false dominates but some real elements remain identifiable".

VISUAL-FABRICATION DISCIPLINE — be EXTRA CAREFUL with seemingly-clean images:
  ReMMDBench includes many high-fidelity AI-generated press-style images that LOOK like real photos.
  When the post images are PRESS-STYLE but the post text claims a SPECIFIC SUB-EVENT (named person + specific number + specific date + specific location) and the retrieved evidence does NOT carry an EXPLICIT fact_check or fact_brief CONFIRMATION of that specific sub-event, BE SKEPTICAL of declaring V1=false. Lack of fact-check is NOT the same as confirmation; consider whether the image's photo-realism is the very signal of AI synthesis.
  Conversely, when the evidence carries an explicit `fact_check` of "image confirmed authentic" or news_snippet quoting the same image source, you may safely keep V1=false.

================================================================================
HOW TO USE YOUR INPUTS
================================================================================

(1) THE POST IMAGES — examine each image carefully. Cross-check what you SEE against what the post text claims AND against the visual descriptions in `image_content` evidence.

(2) RAG MEMORY-BANK EVIDENCE (textual ground truth):
   • `news_snippet` / `fact_brief`: textual ground truth about the event.
   • `image_content`: TEXTUAL DESCRIPTION of what the GROUND-TRUTH images of this topic look like. If post image visually disagrees with `image_content`, that is V2 + likely C2.
   • `fact_check`: official rebuttal. If it labels a specific claim false, that claim is CONTRADICTED.
   • `social_post`: secondary signal.

(3) ATOMIC POINTS (image atoms, cross-modal atoms, text atoms, visual_indicators) — your atomic decomposition from THREE separate qwen passes (image-only, cross-modal, text-only). Use them as a structured map of what to verify.

(4) UPSTREAM ANALYSER HINTS (image_analyser + text_analyser) — these are SOFT priors. Adopt a flagged label only if you can confirm the cue. Reject the prior if the cue is absent or weak.

EVIDENCE DISCIPLINE:
  • Topical-relatedness ≠ confirmation. Evidence "about the same topic" without restating the post's specific claim is UNVERIFIED, not SUPPORTED.
  • A SPECIFIC quantity / quote / attribution in the post needs an EVIDENCE quantity / quote / attribution that matches — otherwise UNVERIFIED or T1.
  • Past-tense references to historical events are NOT contradicted by later evidence; only call CONTRADICTED when evidence directly rebuts the SAME specific claim.

================================================================================
DECISION PROCEDURE (fill the JSON in this EXACT order)
================================================================================

STEP 1 — `core_claim`: ONE short sentence stating the post's central assertion.

STEP 2 — `key_subclaims`: 3–8 SPECIFIC sub-claims from the post (factual: who/what/when/where/how-much; or framing: "X was a decisive victory"). Include image-grounded sub-claims when relevant.

STEP 3 — `subclaim_findings`: for EACH sub-claim, look across ALL evidence + the actual images, and decide its status with a SHORT note:
   {"subclaim": "<text>", "status": "SUPPORTED|PARTIALLY_SUPPORTED|CONTRADICTED|UNVERIFIED", "evidence_id": "<id_or_null>", "note": "<short>"}

   Status definitions:
   • SUPPORTED — evidence (or image) explicitly confirms the sub-claim's specifics.
   • PARTIALLY_SUPPORTED — evidence confirms the topic but DIFFERS on magnitude / details / framing / attribution.
   • CONTRADICTED — evidence states the opposite OR the IMAGE you see contradicts the post text OR a fact_check rebuts the claim.
   • UNVERIFIED — no evidence or pixel can speak to it.

STEP 4 — `image_findings` (REQUIRED when there is ≥1 image): for each image, decide each of {v1_synthetic_present, v2_edit_present, c1_entity_mismatch, c2_context_mismatch, c3_pragmatic_mismatch} as boolean. Also fill `matches_post_claim_about_image` and `matches_evidence_image_content`. Add a one-sentence `explanation` naming the concrete cue if any flag is true.

STEP 5 — `level2_taxonomy`: emit the multi-label list of L2 labels that fired according to the LEVEL-2 GUIDE. Order doesn't matter; emit FULL canonical names ("T1 Fabrication", "V2 Visual Editing", etc.). For posts without images, NEVER emit V1/V2/C1/C2/C3.

STEP 6 — `verdict_reasoning`: 3–5 short bullets summarising your L1 reasoning. Call out:
   - the dominant truth/falsity proportion,
   - which L2 labels fired and why they map to the L1 label,
   - which anti-bias rule applies (e.g. "T1 fired ⇒ NOT Mostly True", "T1+V1 with companions ⇒ False").

STEP 7 — `level1_verdict`: pick the label whose OPERATIONAL DEFINITION best matches your L2 set + findings. Apply the ANTI-BIAS RULES strictly.

STEP 8 — `level3_rationale`: ONE long sentence (25–80 words) in the POST'S language, explaining why the post received its L1 label.

CONSISTENCY RULES (the ONLY hard constraints; enforced automatically downstream):
  • If `level2_taxonomy` is `[]` ⇒ `level1_verdict` MUST be `True`.
  • If `level2_taxonomy` is non-empty ⇒ `level1_verdict` MUST NOT be `True`.
